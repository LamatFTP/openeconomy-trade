### Exchange Flow

**Phase 1: Propose**
```
Icon: Send
Title: Send a Proposal
Description: 
- Select the skill you want from them
- Estimate the hours needed
- System calculates value (hours Ã— their rate)
- Offer what you'll provide in return
- System shows if exchange is balanced
- Add a note explaining what you need
- Submit proposal

Example Proposal:
"I'd like 3 hours of Spanish tutoring ($120 value at your $40/hr rate).
In exchange, I'll provide 2.4 hours of web design ($120 at my $50/hr rate).
I'm building a small business site and want to improve my Spanish for travel."
```

**Phase 2: Negotiate**
```
Icon: Chat
Title: Agree on Terms
Description:
- Recipient reviews proposal
- Can accept, counter, or decline
- Counter-proposals adjust hours or terms
- Continue until both agree
- Once accepted, exchange is "locked in"

Negotiation Example:
Original: 3 hours tutoring for 2.4 hours design
Counter: "Could we do 4 hours tutoring for 3.2 hours design? 
         I have a larger project than expected."
Final: Both agree to adjusted terms
```

**Phase 3: Complete**
```
Icon: Check
Title: Do the Work
Description:
- Provider delivers the service
- Track actual hours (may differ from estimate)
- Communicate throughout
- Provider marks "completed" when done

Tips:
- Be responsive to messages
- Deliver quality work
- Communicate if timelines change
- Document what was delivered
```

**Phase 4: Confirm**
```
Icon: Star
Title: Confirm & Rate
Description:
- Receiver confirms work was completed
- Both parties rate each other (1-5 stars)
- Leave brief review (optional but encouraged)
- Credit balances automatically updated
- Exchange recorded permanently

Rating Guidelines:
5 stars: Exceptional, exceeded expectations
4 stars: Good, would work with again
3 stars: Acceptable, met basic requirements
2 stars: Below expectations, some issues
1 star: Poor experience, significant problems
```

### Questions (5)

**Q1**
```
Q: What if an exchange is unbalanced?
A: The system warns you if values differ by more than 10%. 
   Small imbalances are recorded as credit/debit. 
   You don't need to balance every exchange perfectlyâ€”
   it evens out over time across multiple exchanges.
```

**Q2**
```
Q: What if someone doesn't complete their part?
A: First, try to work it out directly. If that fails, 
   you can raise a dispute. A neutral mediator (Verified+ member) 
   will help resolve it. Repeat offenders lose reputation.
```

**Q3**
```
Q: Can I change my hourly rate?
A: Yes, anytime. Changes apply to new exchanges only. 
   Existing agreements keep their original terms.
```

**Q4**
```
Q: Is this like bartering?
A: Similar concept, but smarter. Traditional barter requires 
   "double coincidence of wants" (you have what I need AND vice versa). 
   OpenEconomy uses time-value normalization, so you can exchange 
   with anyone and balance over time across multiple exchanges.
```

**Q5**
```
Q: Do I need cryptocurrency?
A: No! The blockchain records exchanges for transparency, 
   but you never need to buy, hold, or understand crypto. 
   It works in the background.
```
